CONTEXTO DEL PROYECTO

**Industria:** FoodTech / Personal Cooking Assistant
**Producto:** Landing page de caso de estudio (la app aún no existe, se presenta como si existiera)
**Público:** Personas que quieren cocinar con lo que tienen en casa, reducir desperdicio y seguir dietas específicas sin complicaciones.

**Servicios que ofrece la app:**
1. Generación inteligente de recetas basada en ingredientes disponibles.
2. Filtros dinámicos de dietas personalizadas (Keto, Paleo, Tradicional).
3. Herramientas de optimización de despensa para reducir el desperdicio de alimentos.

ESTRUCTURA DE PROYECTO

Implementa una arquitectura modular:
- /index.html (landing principal)
- /css/styles.css (variables CSS, resets, componentes, secciones, responsive)
- /js/main.js (menú móvil, animaciones scroll, filtrado, interactividad)
- /assets/ (logo, imágenes si las hay)

REQUISITOS TÉCNICOS

1. **HTML5 semántico:** Usa etiquetas apropiadas (header, main, section, article, footer)
2. **CSS3 moderno:** 
   - Variables CSS en :root para toda la paleta
   - Flexbox y Grid para layouts
   - Animaciones y transiciones suaves
   - Media queries para responsive (mobile-first)
3. **JavaScript nativo ES6+:**
   - Menú móvil hamburguesa
   - Scroll suave a secciones
   - Animaciones on-scroll (Intersection Observer)
   - Interactividad de filtros (si aplica)
4. **Accesibilidad:**
   - Atributos ARIA donde sea necesario
   - Contraste de colores WCAG AA
   - Navegación por teclado
   - Alt text descriptivo
5. **Rendimiento:**
   - CSS y JS optimizados
   - Lazy loading si hay imágenes
   - Fuentes con display=swap
   - Sin dependencias externas (excepto Google Fonts)
6. **Responsive:** Breakpoints en 480px, 768px, 968px, 1200px
